## 🔐 암호학 기초
### 대칭 암호화 (Symmetric Encryption)
**암복호화에 사용하는 키가 동일한 암호화 방식**
![symmetric encryption](/images/cs/encryption1.png)
- 한 개의 비밀 키를 통해 데이터를 암호화 → 복호화
- 빠른 암호화 및 복호화 속도를 제공하므로 대량의 데이터를 암호화하는데 효과적
- 사용자 증가에 따른 키 분배와 관리의 어려움, 키가 노출되면 데이터의 보안이 위협   
`ex) DES, AES 등`

### 비대칭 암호화 (Asymmetric Encryption)
**공개키(Public Key)**와 비밀키(Private Key) 한 쌍 사용   
공개키로 암호화 → 비밀키로만 복호화 가능 → 키 관리 쉬움   
`ex) RSA`

### 해시 함수 (Hash Function)
입력이 바뀌면 출력값(해시)이 크게 달라짐   
주로 암호 저장, 데이터 검증 등에 사용   
일방향이라 복호화 불가

## 🚪 인증(Authentication) 및 인가(Authorization) 메커니즘
### PKI (Public Key Infrastructure)
공개키 기반의 디지털 인증서 관리 체계   
인증기관(CA)이 웹사이트나 사용자에게 인증서를 발급 → TLS 등에서 본인 확인에 사용

### OAuth 2.0
인가 프레임워크   
“구글 로그인으로 다른 앱 권한 허용” 같은 상황에 사용   

### OpenID Connect (OIDC)
OAuth 2.0 기반에 인증 레이어 추가(ID 토큰 제공)   
사용자 로그인 시 본인 확인을 책임
